# Additional Notes

## Private Repository Notice

The core implementation is currently private.

This public showcase repository is provided for the hackathon submission so judges can review:

- the product concept
- the architecture
- the validated demo scenario
- the positioning for Track 1

The private repository contains the production implementation and internal integrations.

However, the **`agents`** directory in this repository exposes part of the agent architecture and contains materials describing how the agent operates. This includes documentation and interface references for:

- the agent core structure
- reasoning and execution patterns
- the APIs and tools used by the agent to interact with the browser environment
- interaction mechanisms used during exploratory runs

This allows reviewers to understand the operational design of the agent without exposing the full production infrastructure.

## Why The Core Is Private

The private codebase currently includes active product infrastructure, internal project structure, and non-public implementation details that are not ready for open-source release.

## Reviewability

Even though the core repository is private, the submission is still reviewable through:

- the public architecture description
- the validated demo flow
- the product narrative
- the demo video and screenshots attached to the submission
- the partially exposed agent architecture available in the `agents` directory

Additional private code access can be provided to judges on request if needed.
